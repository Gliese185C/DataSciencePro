	1. С момента передачи информации в аналитический отдел, проходит в среднем сутки до получения рекомендации на выдачу кредита ( средний показатель ожидания ответа достаточно долгий )

	1.1 Система работает следующим образом:
		0 - 29 ---> не выдают кредит
		30 - 69 ---> не может по правилам вернуть такое число ( уходит на этап ручного анализа )
		70 - 99 ---> выдают кредит

	1.2 Есть проблема в системе, со слов Софии если приходит число 42, то ей всё равно и они выдают кредит ( как модель смогла вернуть 42 ? )

	2. Числа полученные моделью машинного обучения приходят намного быстрее чем ручной анализ 

	3. Числа приходят в течении 2-х секунд ( хороший показатель )

	4. Текст после ручного анализа приходит в среднем 3 дня ( максимально долго )

	5. Очень большой процент невозравтов после рекомендаций 70 - 99 от модели ml ( machine learning ), 40%
	
	6. Невозравтов после ручного анализа почти нет
	
	7. 75% клиентов отказываются от одобренных кредитов из-за слишком долгого ожидания ( ручной анализ аналитического отдела )



Аналитический блок:
	
	1. Система работает следующим образом:
		0 - 29 ---> не выдают кредит
		30 - 69 ---> не может по правилам вернуть такое число ( уходит на этап ручного анализа )
		70 - 99 ---> выдают кредит

	2. 50% от всех запросов на кредит, модель ml выдаёт числа больше 70 ( рекомендует дать кредит )
	
	3. 20% от всех запросов на кредит, модель ml выдаёт числа меньше 30 ( рекомендует отказать )

	4. 30% от всех запросов на кредит, модель ml сомневается и отправляет запрос на второй этап ручного анализа

	5. Модель оценивает заёмщика в 1 - 2 секунды ( хороший показатель )

	6. Ручной анализ запросов на кредит происходит в течении 2 - 3 дней ( очень долгий показатель )

	7. 66% запросов на кредит одобряют, 33% отказывают 


Если по этим данным решить небольшую задачу с 1000 клиентами, то мы получаем печальный результат:

1000 клиентов отправили запрос на получение кредита ( предположим, что это 10000 лей под 10%, сумма возврата 11000 лей )

50% модель ml одобряет, это 500 клиентов {
	
	из них 40% невозвращают кредит, 60% возвращают, это получается 300 - вернут, 200 - не вернут

	банк получит 300 * 11000 --> 3 300 000
	банк потеряет 200 * 10000 --> 2 000 000

}

20% модель ml отказывает, это 200 клиентов

30% модель ml сомневается, это 300 клиентов, они уходят на ручной анализ {
	
	по данным аналитического отдела 66% одобряют, это 200 клиентов 
	33% отказывают , это 100 клиентов 

	также мы знаем, что 75% из одобренных кредитов ручного анализа отказываются, это 150 отказов
	25% получают, это 50 клиентов
	
	то есть из 300 клиентов ушедших на ручной анализ, получат только 50 ( это очень ужасный результат, есть проблема в аналитическом отделе и она связана с долгим времен ожиданием ответа )
	
	также мы знаем, что возврат кредитов 100%, то есть банк получит 50 * 11000 --> 550 000	
}

Всего было одобрено 550 кредитов из 1000 ( 55% одобренных, 45% неодобренных, плохой показатель )
Банк потратил 550 * 10000 --> 5 500 000
Банк получил обратно 3 850 000
Банк ушёл в минус на 5 500 000 - 3 850 000 --> 1 650 000 лей

То есть банк теряет 30% своих от средст, которые выделяют под выдачу кредита

Заключение: Выдачу кредитов приостановить до исправления ошибок в системе банка 

На мой взгляд, огромная проблема заключается в моделе ml, которая по словами директора МУ довольно старая
Также очень большой процент клиентов уходит на ручной анализ и очень большой процент отказываются
